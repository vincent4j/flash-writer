# 实际案例：从零写一篇演讲稿

基于真实写作会话，为 AI 工具 OpenClaw 创作 30 分钟线下演讲逐字稿的全过程。

## 初始设置

用户创建了这个目录：

```
0305 OpenClaw 线下分享/
├── 写作思路.md          # 写作模板（已填写）
├── 会议日程.md          # 约束文档 - 会议日程
├── references/
│   ├── 01-用OpenClaw搭建一个7x24帮你赚钱的AI员工.md
│   └── 02-2万字Openclaw保姆教程.md
└── assets/              # 空目录，写作时添加截图
```

## 写作思路（关键部分）

```markdown
## 原则
这是一篇命题文章的撰写，文章的目录请看「会议日程」文档里，
可以适当调整目录，但是不能改的面目全非。
本文章将用于本次会议上的演讲的逐字稿。

## 受众
很多人都听说过 OpenClaw，但仅仅停留于此。
部分人尝试过安装但没成功；也有部分人看到安装文档就被吓退了。

## 目标
参加会议的所有人都能安装起来，实际操作一个案例，
心里惊喜：哇，终于算是用起来了。

## 写作思路
1、先科普 OpenClaw
— OpenClaw 的定位？
— OpenClaw 是什么？

2、OpenClaw 和大家之前知道的工具的差异
对比 Claude Code 和 ChatGPT

3、案例：24 小时全自动热点收割机
》参考 01 文章的案例，用大白话重新讲

4、更多应用场景展示
》一带而过，重数量（4-8 个），贴近受众

5、安装指南：手把手搭建
》参考 02 文章，简化步骤，只保留最短路径。
  统一用阿里云百炼 Coding Plan，买最便宜的套餐。

## 格式偏好
- 标题不加序号
- 对比类内容用表格
- 每个模块末尾用「一句话」金句收尾
- 标题用 emoji 装饰

## 写作风格
口语化，像跟朋友聊天。喜欢用类比解释概念。
段落短，节奏快。常用反问句引导。
```

## 第一步：AI 阅读所有内容并对齐

AI 阅读了写作思路、会议日程（约束）和两篇参考文章，然后呈现：

> **我理解的任务：**
> - 为一个 3 小时线下活动写 OpenClaw 演讲逐字稿
> - 受众：听说过但没装成功的人
> - 目标：所有人都能装上并用起来
> - 约束：大纲遵循会议日程（上半场：介绍+案例，下半场：更多案例+安装）
> - 参考 01：热点收割机案例（用于第 3 节）
> - 参考 02：2 万字安装教程（用于第 5 节）
> - 第 1、2、4 节没有"参考"标注——我来提供原创内容
>
> **有遗漏的地方吗？**

## 第二步：AI 推荐大纲

```
OpenClaw 是什么？
  - "住在你电脑里的数字员工" 类比
  - 它怎么工作的：能看屏幕、操作浏览器

和 ChatGPT、Claude Code 有什么区别？（思路明确指定这两个）
  - 表格：ChatGPT = 顾问、Claude Code = 没手的程序员、OpenClaw = 全能员工

用 OpenClaw 打造"24小时全自动热点收割机"（借鉴参考 01）
  - 为什么做：内容创作者的痛点
  - 4 步搭建流程
  - 执行过程和结果截图

更多应用场景（我的建议，6 个贴近受众的案例）
  - 竞品监控助手 - 每日竞品报告
  - AI 早报助手 - 自动新闻摘要
  - 社群精华整理 - 聊天记录摘要
  - 批量生成小红书图文 - 批量内容创作
  - 网页数据抓到表格 - 网页抓取到 Excel
  - 自动回复客户常见问题 - FAQ 自动回复

安装指南：手把手搭建（借鉴参考 02，简化）
  - 只要 5 步：买 API Key → 装 Node.js → 运行向导 → 设置人设 → 对接飞书
```

## 第三步：用户反馈（多轮）

**第 1 轮 - 用户说：**
> "竞品监控" 太简单了，换成更有冲击力的。案例按这个顺序：批量重命名文件、网页数据抓到表格、定时发送日报提醒、电商平台比价、知乎批量收藏回答、GitHub 查看 Issues

**AI 立即调整** —— 不争论，严格按照指定顺序。

**第 2 轮 - 用户说：**
> "对比 Claude Code 和 ChatGPT" - 你写的对比不够清晰，"得自己跑" 是什么意思？

**AI 重写**，更清楚的解释："它帮你写代码，但代码写完之后，你得自己复制、自己运行、自己调试"

**第 3 轮 - 用户说：** "开始写"

AI 进入写作阶段。

## 第四步：边写边反馈

AI 逐节写作。第一节写完后确认风格是否匹配。

**写作过程中的关键时刻：**

1. **风格验证**：写完第 1 节后，AI 问"风格对吗？"用户说语气不对。AI 请用户提供风格样本，重新提取特征，调整写法。

2. **截图处理**：写安装指南时，AI 用浏览器工具导航到飞书开发者平台，截取应用创建流程的截图，保存为 `assets/OpenClaw线下分享演讲稿/飞书-创建应用.png`。

3. **主动检查**：写完所有章节后，AI 发现热点收割机章节引用了执行过程截图但还没有。AI 主动问："第 3 节提到了执行过程——需要我也截这些图吗？"

## 最终成果

一篇 360 行的演讲逐字稿：
- 对话式语气，匹配用户的风格
- 台下互动环节（"举个手看看"、"有人说说吗？"）
- 所有对比都用表格
- 每节末尾都有一句话金句
- 9 张截图放在正确位置
- 实用的、匹配受众水平的案例

## 经验教训

这些已经融入 SKILL.md 的规则中，但值得强调：

| 出了什么问题 | 技能现在怎么防止 |
|---|---|
| AI 照搬参考文章而不是重写 | "你是代笔人，不是复印机" 核心原则 |
| AI 给非技术受众推荐了科幻级用例 | "匹配受众水平" 规则 |
| AI 私自替换了对比对象（用 Manus 替代 ChatGPT） | "大纲中确认细节" 规则 |
| 思路没指定时 AI 不主动提供案例建议 | "没有标注 = 由你决定" 指令 |
| 风格不对直到用户抱怨才发现 | "风格验证（仅第一节）" 步骤 |
| 图片作为独立最后一步处理，导致返工 | "写作过程中处理图片" 的内联方式 |
