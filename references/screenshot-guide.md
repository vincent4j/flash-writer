# 截图指南

当写作思路中指定"AI 截图"时，使用浏览器自动化工具截取相关页面的干净截图。

## 如何截图

1. **导航**到讨论中的实际产品/工具/页面
2. **准备页面状态** —— 如需登录先登录，导航到正确的页面，展开相关面板
3. **截取** —— 使用浏览器工具（Playwright、Chrome DevTools 等）
4. **保存**到 `assets/<文章名>/`，用描述性文件名

## 命名规范

用描述性名称，能看出内容：
- `飞书-创建应用.png`（不是 `截图-1.png`）
- `API密钥-设置页.png`（不是 `image-02.png`）
- `机器人-成功回复.png`（不是 `capture-2026-03-05.png`）

## 在文章中引用

```markdown
![](assets/<文章名>/飞书-创建应用.png)
```

## 需要登录的页面

如果页面需要登录：
1. 提醒用户："我需要截取 [页面] 的截图，但需要登录。你能帮我登录一下吗？"
2. 等用户确认
3. 然后导航并截取

## 无法截图时

如果截图不成功（浏览器工具不可用、页面被拦截等）：
- 插入描述性占位符：`![TODO: 飞书创建机器人页面截图]()`
- 告诉用户需要什么截图以及放在哪里

## 注意事项

- 窗口大小设置合理（1280x800 或类似）
- 避免截到个人隐私信息
- 如果页面有加载状态，等内容完全加载后再截取
