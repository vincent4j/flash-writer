description: "初始化闪电写手工作环境，引导用户完成写作思路，进入内容创作。"
---

调用 flash-writer 技能，但不要直接进入四步工作流。按以下步骤执行：

---

## 第一步：介绍 + 询问目录

先向用户输出以下内容（原样输出），然后等待用户回复：

```
闪电写手 ⚡ 已启动

只需按模板填写你的写作思路，我来引导你快速完成创作——降低写作难度，提升写作效率。

流程：填写写作思路 → 校验完备性 → 对齐信息 → 推荐大纲 → 确认大纲 → 写正文 → 四层自检 → 确认完稿

我会在你的项目中创建或复用唯一的 `references/` 文件夹，准备写作模板和参考案例。

默认放在项目根目录，你也可以指定其他目录。直接回车用默认位置，或输入路径：
```

等待用户回复。用户回车或不指定路径，则用项目根目录；用户指定路径则用指定路径。

---

## 第二步：创建文件 + 引导填写

用户项目始终只使用一个 `references/` 目录：

- 目录不存在时创建；已经存在时直接复用
- 分别检查 `writing-template.md`、`real-example.md` 和 `screenshot-guide.md`
- 只复制缺失文件；同名文件已经存在时保留原文件，绝不覆盖，也不创建第二个目录

使用本技能目录中的 `scripts/profile_store.py init --project-dir <用户选择的目录>` 初始化个性化写作档案。该命令只创建缺失文件，不覆盖已有档案；项目状态保存在当前用户的 `~/.flash-writer/projects/`，不会在用户选择的目录中创建 `.flash-writer/`。不要把个人规则写入 Skill 本体。

文件准备完成后，向用户输出以下内容；每个文件根据实际情况标注“已创建”或“已保留”：

```
已准备 `references/`：

- [writing-template.md](<实际绝对路径>) — 已创建/已保留，填写写作思路
- [real-example.md](<实际绝对路径>) — 已创建/已保留，完整案例
- [screenshot-guide.md](<实际绝对路径>) — 已创建/已保留，图片截图指南

同时已启用个人写作档案。后续写作中确认沉淀的可复用经验会自动延续，不需要在下一篇重复说明。

写作风格留空时会自动使用 Flash Writer 内置默认风格；你也可以直接描述自己的风格，或提供自己写过的文章作为样本。

请打开 [writing-template.md](<实际绝对路径>)，按提示填写受众、目标、写作思路和参考资料。

写完后回来告诉我「已写完」。
```

注意：把每个 `<实际绝对路径>` 替换成用户所选目录下对应文件的绝对路径；路径包含空格时用尖括号包裹链接目标。文件名使用 markdown 链接格式，让用户可以直接点击打开。不要输出不存在的动态目录名。

不要自动进入下一步，等待用户回复「已写完」或类似表述。

---

## 第三步：校验写作思路

用户说「已写完」后：

1. 读取用户填写的 `writing-template.md`（或用户指定的思路文件）
2. 校验完备性，逐项检查：
   - **原则**：是否填写了内容类型和硬约束
   - **受众**：是否明确了目标读者和认知水平
   - **目标**：是否用具体场景描述了预期效果（而非抽象的"了解 X"）
   - **写作思路**：是否有章节级大纲
   - **内容参考**：是否提供了参考来源（URL 或本地文件）
3. 检查是否存在矛盾（如：受众是非技术人群，但写作思路要求深入讲源码）
4. 输出校验结果：
   - 如有缺失或矛盾，列出具体问题，请用户补充修改后再次告知「已写完」
   - 如无问题，输出：

```
写作思路校验通过，没有缺失或矛盾。开始内容创作。
```

然后正式进入 SKILL.md 中定义的四步工作流（从第一步「对齐信息」开始）。

内置默认风格保存在 Skill 的 `references/built-in-style.md` 中，由工作流按需读取；不要复制到用户项目，也不要覆盖用户提供的风格文件。
