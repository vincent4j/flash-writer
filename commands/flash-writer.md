description: "初始化闪电写手工作环境，引导用户完成写作思路，进入内容创作。"
---

调用 flash-writer 技能，但不要直接进入四步工作流。按以下步骤执行：

---

## 第一步：介绍 + 询问目录

先向用户输出以下内容（原样输出），然后等待用户回复：

```
闪电写手 ⚡ 已启动

只需按模板填写你的写作思路，我来引导你快速完成创作——降低写作难度，提升写作效率。

流程：填写写作思路 → 校验完备性 → 对齐信息 → 推荐大纲 → 确认大纲 → 写正文

我会在你的项目中创建一个 `references/` 文件夹，存放写作模板和参考案例。

默认放在项目根目录，你也可以指定其他目录。直接回车用默认位置，或输入路径：
```

等待用户回复。用户回车或不指定路径，则用项目根目录；用户指定路径则用指定路径。

---

## 第二步：创建文件 + 引导填写

检查用户指定目录下是否已存在 `references/` 目录：
- 如果不存在，创建 `references/` 并复制文件
- 如果已存在，改为创建 `writer-references/` 并复制文件，同时在输出中将文件夹名替换为 `writer-references`

将本技能的 `references/` 目录中的文件复制到实际创建的目录下。

文件创建完成后，向用户输出以下内容（原样输出）：

```
已创建 `references/`，包含 3 个文件：

- `writing-template.md` — 写作模板，填写你的写作思路
- `real-example.md` — 一个完整案例，供参考
- `screenshot-guide.md` — 图片截图指南

请打开 [references/writing-template.md](references/writing-template.md)，按提示填写你的受众、目标、写作思路、参考资料等。

写完后回来告诉我「已写完」。
```

注意：文件名使用 markdown 链接格式，让用户可以直接点击打开。

不要自动进入下一步，等待用户回复「已写完」或类似表述。

---

## 第三步：校验写作思路

用户说「已写完」后：

1. 读取用户填写的 `writing-template.md`（或用户指定的思路文件）
2. 校验完备性，逐项检查：
   - **原则**：是否填写了内容类型和硬约束
   - **受众**：是否明确了目标读者和认知水平
   - **目标**：是否用具体场景描述了预期效果（而非抽象的"了解 X"）
   - **写作思路**：是否有章节级大纲
   - **内容参考**：是否提供了参考来源（URL 或本地文件）
3. 检查是否存在矛盾（如：受众是非技术人群，但写作思路要求深入讲源码）
4. 输出校验结果：
   - 如有缺失或矛盾，列出具体问题，请用户补充修改后再次告知「已写完」
   - 如无问题，输出：

```
写作思路校验通过，没有缺失或矛盾。开始内容创作。
```

然后正式进入 SKILL.md 中定义的四步工作流（从第一步「对齐信息」开始）。
